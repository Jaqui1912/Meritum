# Implementación de Verificación de Correo (Backend)

Atención IA encargada del Backend en ASP.NET Core: El objetivo de esta tarea es implementar un sistema de verificación de correo electrónico para los nuevos usuarios de la API de Meritum.

A continuación, se define el flujo esperado y la plantilla oficial para el envío de correos.

## 1. Cambios a la Base de Datos (Migración requerida)

Debes agregar dos campos nuevos a la entidad

User (o equivalente):

* `IsVerified` (bool): Por defecto `false`. Indica si el correo pertenece a su dueño.
* `VerificationToken` (string nullable): Un hash alfanumérico único generado al momento del registro. (Opcional: puedes usar JWTs con límite de tiempo, ej. 24 horas).

## 2. Modificaciones al Flujo de Autenticación (

AuthController.cs)

### Endpoint de Registro (`[HttpPost("register")]`)

* La lógica actual no debe cambiar en esencia (sigue creando al usuario en la BD).
* **Adición** : Al registrar al usuario, genérale un `VerificationToken` y guarda `IsVerified = false`.
* **Servicio SMTP** : Conéctate a un servicio SMTP (Google SMTP, SendGrid, etc.) y envíale de manera asíncrona un correo electrónico a la dirección que proporcionó.
* Este correo debe inyectar la  **plantilla HTML proporcionada más abajo** , reemplazando la variable `{{VERIFICATION_URL}}` con una URL real construida por tu servidor. Ejemplo: `https://meritum.koyeb.app/api/auth/verify?token=XYZ...`

### Nuevo Endpoint de Verificación (`[HttpGet("verify")]`)

* Recibe el `token` de verificación vía Query String.
* Busca en la BD al usuario que posea este `VerificationToken`.
* **Si lo encuentras** : Cambia su `IsVerified` a `true` y limpia el `VerificationToken` (pasarlo a `null`).
* **Retorno** : Devuelve un HTML básico confirmando el éxito de la verificación o redirige a una URL de éxito en caso de tener un frontend web.

### Endpoint de Login (`[HttpPost("login")]`)

* Busca al usuario con sus credenciales normales.
* **Validación Crítica** : Antes de devolver el Ok() y el JWT, evalúa si `user.IsVerified == true`.
* **Si es false** : Devuelve un `Unauthorized` (`401`) u otro error claro (`403`) especificando `"message": "Debes verificar tu correo electrónico antes de ingresar."`
