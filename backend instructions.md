# Instrucciones para agregar Video de Presentación en la Web

## Contexto

Se agregó un nuevo campo al backend: **`PreviewVideoUrl`** — un video corto de ~10 segundos que se muestra como preview en la galería de la app móvil (estilo Netflix). Este video es **independiente** de los videos de funcionalidad del proyecto.

ProjectsController.cs

* **Create** : Si `dto.PreviewVideoFile` no es null, lo guarda en la carpeta `previews/` y asigna la URL a `PreviewVideoUrl`
* **Update** : Si `dto.PreviewVideoFile` no es null, reemplaza el preview existente. Si `dto.RemoveExistingPreviewVideo` es true, limpia el campo
* **GetAll/GetById** : Transforma la URL relativa a absoluta (`baseUrl + PreviewVideoUrl`)

### 3. Endpoints de la API (referencia)

| Acción | Método  | URL                                | Campo del FormData                                                     |
| ------- | -------- | ---------------------------------- | ---------------------------------------------------------------------- |
| Crear   | `POST` | `/api/projects?adminId=xxx`      | `PreviewVideoFile` (archivo)                                         |
| Editar  | `PUT`  | `/api/projects/{id}?adminId=xxx` | `PreviewVideoFile` (archivo) + `RemoveExistingPreviewVideo` (bool) |

### 4. Notas importantes

* El archivo se guarda en la carpeta `uploads/previews/` del servidor
* Solo se acepta **1 archivo** (no es lista como los videos de funcionalidad)
* El video debe ser corto (~10 segundos) — considerar agregar validación en el frontend
* El campo es **opcional** — los proyectos existentes no se ven afectados
